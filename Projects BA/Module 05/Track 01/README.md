## What this is

A complete Lean Six Sigma DMAIC engagement for **DistributorCo**, a global multi-category e-commerce distributor (~$1.2B annualized revenue, 12,000+ SKUs across 50 categories, 5 regional fulfillment centres). The engagement diagnoses fulfillment quality, computes Statistical Process Control limits, optimizes inventory via EOQ + Safety Stock, and locks the gains via a per-mode Control Plan.

The hero finding is uncomfortable: **three of four shipping modes are statistically out of control** (Cpk ≤ 0). One — First Class — is deterministically late by 1 day on every single order (σ = 0, Cpk undefined). This is a process-design failure, not a variance problem.

## Methodology notes

### SPC charts

- **X-bar / R charts** built per mode using daily subgroups (n ≈ 5–15 orders/day per mode). Constants A2, D3, D4 from Montgomery, *Introduction to Statistical Quality Control* (8th ed.), Table VI.
- **p-charts** for late-delivery proportion: 3σ binomial limits with `σ_p = √(p̄(1−p̄)/n̄)`.
- **Western Electric rules** applied to X-bar series:
  - R1: any point beyond 3σ
  - R2: 2 of 3 consecutive beyond 2σ (same side)
  - R3: 4 of 5 consecutive beyond 1σ (same side)
  - R4: 8 consecutive on the same side of the centerline
- **Cp/Cpk capability indices** computed against scheduled-days as USL, zero as LSL. When σ = 0 (First Class), Cpk is mathematically undefined; we classify the process as "DETERMINISTIC" rather than emitting a meaningless value.

### EOQ / Safety Stock

- **EOQ** via classical Wilson formula: `Q* = √(2DS/H)`, where D = annual demand (units), S = order processing cost ($50/PO industry-standard), H = annual holding cost = `H_rate × unit_price` with `H_rate = 0.20` (industry-standard 20%).
- **Safety Stock** at 95% service level: `SS = Z × σ_D × √L`, with Z = 1.645, σ_D = daily-demand standard deviation, L = average lead time (days).
- **Reorder Point**: `ROP = (D/365) × L + SS`.
- Comparison baseline assumes monthly ordering (Q_current = D/12), which yields the headline savings of $50,629/yr across the top 10 SKUs.

### OEE

- **OEE = Availability × Performance × Quality** (industry-standard form).
- **Availability** = 1 − cancel-rate (proxy: orders not in CANCELED state).
- **Performance** = average ratio of `scheduled_days / real_days`, clipped to [0, 1].
- **Quality** = 1 − late-delivery rate.

## Honest call-outs

These are deliberate analytical choices and disclosures that a reviewer should understand before reading the deliverables.

1. **DistributorCo is a synthetic anchor.** Per-order numbers are real (180,519 orders from the actual DataCo dataset), but the firm-level $1.2B revenue and 12K-SKU narration are *scaled for boardroom relevance*. The dataset describes 118 distinct SKUs across 5 markets — sufficient for SPC and EOQ analysis, but small relative to a real-world distributor at $1B scale. The narration is honest about this.

2. **First Class shipping has σ = 0** across all 27,814 orders (every order ships in exactly 2 days). This is not realistic for real-world carriers; the dataset has an engineered/synthetic component on the shipping-duration field. We've handled this honestly:
   - Cpk is correctly classified as UNDEFINED (not silently emitted as zero)
   - The "DETERMINISTIC_PROCESS" status is documented
   - Reaction plan is "PROCESS DESIGN FAILURE — renegotiate SLA or change carrier" rather than "reduce variance"
   - The narrative reframes this as a *richer* DMAIC story (design failure beats variance problem)

3. **The "Bullwhip Effect" in the original brief is reframed** as **single-tier demand-signal distortion across the order-to-delivery pipeline**. The dataset captures only the distributor → customer tier; classical multi-tier Bullwhip (manufacturer → distributor → retailer → customer) cannot be measured. The single-tier proxy is the *same statistical phenomenon* (variance amplification from information lags) at one tier of the supply chain.

4. **EOQ assumes monthly baseline ordering.** Actual current ordering frequency is unknown; if it's worse than monthly (which is likely given the chronic stockout pressure), real savings will be higher. The $50,629 figure is therefore conservative.

5. **Holding cost rate (20%)** is industry-standard (covers capital cost, storage, obsolescence, shrinkage, insurance) but firm-specific. Adjust in `InventoryOptimizer.xlsx` cell `Inputs!C6` to test sensitivity.

6. **SPC limits assume process stability.** If the underlying process changes (new carrier, new payment gateway, new fulfillment centre), the limits should be recomputed via `spc_monitor.py`. The Control Plan recommends weekly Quality Council review.

7. **Margin erosion ($0.78/late order) is from-data only.** Hidden costs — customer service tickets, refunds, returns, lifetime-value impact — are not in the dataset. Real margin erosion is conservatively 5–10× higher; the deck flags this explicitly.

The deliberate choice to re-interrogate the same operational dataset through complementary frameworks is itself the portfolio statement: **good analysts look at the same data multiple ways, choosing the lens that matches the question being asked.**

## How to extend

- **More SKUs**: change the top-N filter in the notebook's IMPROVE section (currently `head(10)`). EOQ math scales linearly.
- **Real holding cost**: replace `H_rate = 0.20` with firm-specific data (CFO will provide).
- **Real order processing cost**: replace `S_cost = 50` with actual operations data.
- **Replace "monthly ordering" baseline**: the `current_cost` formula in `EOQ_Calc` assumes Q = D/12. If you know the actual order quantity per SKU, parameterize it.
- **Add a 5th shipping mode**: extend the `cap_data` table in `SPC_Limits` and add a new row to the Control Plan.
- **Schema Registry equivalent**: if migrating to a streaming SPC monitor, the `dim_*` tables become reference data and `fact_orders` becomes the event stream.
